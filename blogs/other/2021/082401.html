    frp实现内网穿透

    ByoneJie 2021-8-24 frp

    # 准备工作

    • 准备一台有公网IP的服务器作为frp的服务端
    • 准备一台本地的win或者liunx都行做为客户端
    • 根据自己的系统下载对应的frp（下载链接 (opens new window)）
    • 本文配置用于通过自定义域名访问部署于内网的 web 服务

    # 什么是frp

    frp 是一个可用于内网穿透的高性能的反向代理应用，支持 tcp, udp 协议，为 http 和 https 应用协议提供了额外的能力，且尝试性支持了点对点穿透。 详情文档查看（官方文档 (opens new window)）

    # 配置服务端

    SSH连接上服务端服务器 进入目录（根据自己喜好把文件放在哪）

    cd /usr/local
    1

    创建一个frp的目录并进入该目录

    mkdir frp
cd frp
    1
    2

    使用wget指令下载frp。

    wget 下载链接
    1

    使用tar指令解压tar.gz文件并进入解压好的目录

    tar 解压的文件名
ls #查看当前目录文件
cd 解压好的文件夹名
    1
    2
    3

    进入目录发现目录有些客户端的文件可以删掉

    rm -f frpc
rm -f frpc.ini
    1
    2

    使用vi命令编辑frps.ini文件(vi命令详情查看文档 (opens new window))

    vi frps.ini
    1

    配置操作（记得服务器安全组开启端口）

    # frps.ini
[common]
bind_port = 7000 //需要和客户端绑定的端口
vhost_http_port = 8080 //使用域名访问web时带上的端口
    1
    2
    3
    4

    客户端的配置到这就结束了，接下来让服务端frp在后台运行起来

    #记得要先进入到frp的目录
nohup ./frps -c ./frps.ini &
    1
    2

    # 配置客户端

    我是用的是win系统所以要去下载win的frp（如果是liunx则和配置服务端差不多）
    下载下来后把多余的服务端文件删掉frps frps.ini这两个文件
    打开frpc.ini配置

    [common]
server_addr = 服务端公网ip 
server_port = 7000  #服务端绑定的端口
admin_addr = 127.0.0.1  #本地可视化后台管理ip（想要的话可以设置，不需要则可以省略admin配置）
admin_port = 7400  #本地后台访问端口
admin_user = admin #本地后台账号
admin_pwd = admin #本地后台密码

[web]    #指定web配置
type = http #http类型
local_ip = 127.0.0.1 #本地ip
local_port = 8080 #本地服务的端口
custom_domains = 域名  #访问web的域名（必须）

    打开cmd进入到frp目录运行

    frpc -c frpc.ini   #frpc.ini可以创建多个配置文件，只需运行的时候访问你的:配置名.ini

    这时客户端也配置好了，只需要开启本地运行的项目通过域名加上端口号即可映射到外网访问
