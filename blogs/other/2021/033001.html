    Git上传项目到github

    ByoneJie 2021-3-30 GitGitHub

    # Git下载安装步骤

    # 1.打开git官网下载

    官网 (opens new window)下载Git时，速度几乎是超不过20KB，解决方法有很多，下载直接贴出git的下载地址。 (git新下载地址 (opens new window))直接找到系统对应的版本看日期选择最新版本下载即可

    # 2.下载完成选择安装

    打开安装包，选择好安装的目录，其他默认下一步即可。

    # 3.安装完成

    双击打开git-bash.exe文件会跳出一个git的命令窗口

    # 使用Git上传项目

    # 配置Git

    打上下面两段命令配置一下用户名和邮箱，如果一开始不配置在上传项目提交注释时就会叫你填写用户名和邮箱，那时候配置可能明明跟着打上了但还是会继续提示。如果上传项目时出现叫你填写用户名和邮箱的建议删除本地仓库，一开始就配置下面两段命令。

    git config --global user.name "用户名"
git config --global user.email "自己的邮箱@qq.com"

    删除本地仓库命令

    #显示当前目录文件及文件夹
ls -a
#然后删除git目录
rm -rf .git
#最后再来一个命令就干净了
rm -rf foldername

    # 初始化Git项目

    进入目录初始化项目

    cd ###
git init
    1
    2

    # 提交项目到本地仓库

    将项目上所有的文件添加到仓库中
    如果想添加某个特定的文件，只需把.换成这个特定的文件名即可。

    #注意add后面空格加一个点
git add .
    1
    2

    注意！！！ 添加的时候可能会出现下面那段警告
    warning: LF will be replaced by CRLF in .....
    The file will have its original line endings in your working directory
     这个不用管的，直接下一步即可（问题出在不同操作系统所使用的换行符是不一样的【详细解答 (opens new window)】）
    要取消这个警告只需执行下面命令（建议开启）

    #false 关闭   true开启
git config core.autocrlf false
    1
    2

    # 提交项目到Git远程仓库

    下面命令表示对这次提交的注释，双引号里面的内容可以根据个人的需要改

    git commit -m "根据自己修改内容"
    1

    将本地的仓库关联到github上（首先在github上有一个仓库，没有则进屋新建仓库复制链接）

    git remote add origin 仓库地址
    1

    把代码上传到github仓库

    #master为仓库的分支名
git push -u origin master
    1
    2

    执行完后，如果没有异常，会等待几秒，然后跳出一个让你输入Username和Password 的窗口，你只要输入github的登录账号和密码就行了。

    登录成功则会出现上传文件进度的东西就表示成功了。
